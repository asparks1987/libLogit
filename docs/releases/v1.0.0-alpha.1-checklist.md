# v1.0.0-alpha.1 Release Candidate Checklist

Use this checklist to promote the draft Alpha notes into a final release
candidate. Every checked item should have command output, a hosted CI link, or a
file diff as evidence. Current local evidence is recorded in
[v1.0.0-alpha.1 Release Candidate Evidence](v1.0.0-alpha.1-rc-evidence.md).

## Local Preflight

- [ ] Confirm the worktree contains only intended Alpha release changes.
- [x] Run the repeatable local release-candidate verifier:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify-rc-local.ps1
```

- [ ] After committing the intended release-candidate changes, run the strict
  clean-worktree release-candidate verifier:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify-rc-local.ps1 -RequireCleanWorktree
```

- [x] Run the full local verifier:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify-alpha.ps1
```

- [x] Confirm `scripts/verify-alpha.ps1` passed Python tests, static checks
  including PowerShell and JavaScript script syntax parsing, JavaScript/npm
  source tests, npm pack dry-run, installed npm tarball smoke, C#/NuGet, Go,
  Java Maven, Kotlin Maven, C, C++, CMake, CTest, install, and
  installed-consumer checks.
- [x] Confirm `scripts/verify-rc-local.ps1` passed package metadata sanity and
  JavaScript release artifact smoke for packed tarball metadata, installed
  package metadata, console, local file, remote-path-as-file, and config-loaded
  file output, temp-copy C# NuGet artifact build/inspection for exact nupkg
  name, `.nuspec` metadata, README, and `lib/net10.0/LibLogit.dll`, then
  temp-copy Java Maven jar build/inspection for exact jar name, runtime
  classes, embedded POM, and Maven properties metadata, then temp-copy Python
  wheel/source-distribution inspection plus installed-wheel smoke coverage for
  console, local file, remote-path-as-file, SQLite, and the viewer CLI without
  dirtying the worktree, and rejects a failed full Alpha verifier process
  before artifact promotion.
  Development-mode `-SkipAlphaMatrix` runs the static release gates before
  package metadata and release artifact checks.
- [x] Confirm `.github/workflows/alpha-matrix.yml` validates all v2 config
  examples, bundled samples, shared fixture configs, package metadata, split
  Python wheel modules, sdist release contents including release scripts,
  installed Python wheel behavior through the shared smoke script, and
  PowerShell and JavaScript script syntax.
- [x] Confirm the local static verifier checks hosted workflow release-gate
  parity: required job names match `scripts/check-hosted-ci.ps1`, and hosted
  Python/JavaScript installed-package smoke gates remain wired into the
  workflow.
- [x] Confirm `docs/burndown.md` has the latest readiness percentages and any
  blockers found during verification.
- [x] Confirm `docs/releases/v1.0.0-alpha.1.md` lists features, install notes,
  migration notes, limitations, and beta-track gaps.

## Hosted CI

- [ ] Push the release-candidate branch or open/update the release PR.
- [ ] Confirm the strict clean-worktree verifier passed on the exact commit that
  will be pushed or tagged.
- [ ] Observe `.github/workflows/alpha-matrix.yml` on GitHub Actions after the
  hosted workflow parity changes are pushed.
- [ ] Run the hosted CI check from an authenticated local environment:

```powershell
gh auth login
$head = (git rev-parse HEAD).Trim()
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-hosted-ci.ps1 -HeadSha $head
```

- [ ] Confirm the hosted helper reports every required job as passed:
  `Schema and config examples`, `Static checks`, `Python`, `JavaScript`, `C#`,
  `Go`, `Java`, `Kotlin`, `C`, `C++`, and `Native CMake package`.
- [ ] Record the successful hosted workflow URL, head SHA, and required-job
  summary in `docs/burndown.md`.

## Artifact Sanity

- [x] Confirm Python wheel and source distribution build from the repository
  root and include `py.typed`, bundled configs, schemas, examples, and
  `liblogit-viewer`; the source distribution also includes release docs,
  conformance fixtures, and the full release verification script set; the
  wheel/sdist filenames and internal metadata match the Alpha release name.
- [x] Confirm the built Python wheel installs into an isolated environment and
  immediately logs to console, local file, remote-path-as-file, and SQLite, then
  prints filtered rows through the installed `liblogit-viewer` script.
- [x] Confirm C and C++ install/export works through CMake and that an installed
  consumer can `find_package(libLogit CONFIG REQUIRED)`.
- [x] Confirm C++ users have both a root-header vendoring path and an installed
  `<liblogit/logit.hpp>` migration path documented.
- [x] Confirm C#, JavaScript, Java, Kotlin, and Go package metadata matches the
  Alpha artifact names in the release notes.
- [x] Confirm the C# NuGet artifact builds as `LibLogit.1.0.0-alpha.1.nupkg`
  and includes matching `.nuspec` metadata, README, and
  `lib/net10.0/LibLogit.dll`.
- [x] Confirm the Java Maven artifact builds as `liblogit-1.0.0-alpha.1.jar`
  and includes runtime classes plus matching embedded POM and Maven properties
  metadata.
- [x] Confirm JavaScript users can install the packed npm tarball into a fresh
  consumer project and immediately log to console, local file,
  remote-path-as-file, and config-loaded file output; the npm pack metadata,
  tarball name, and installed package metadata match the Alpha release name.
- [x] Confirm package metadata is aligned to `v1.0.0-alpha.1` release naming
  across Python (`1.0.0a1`), C#/JavaScript/Java/Kotlin (`1.0.0-alpha.1`), and
  native C/C++ CMake package config (`1.0.0`).
