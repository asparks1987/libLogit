# v1.0.0-alpha.1 Release Candidate Checklist

Use this checklist to promote the draft Alpha notes into a final release
candidate. Every checked item should have command output, a hosted CI link, or a
file diff as evidence.

## Local Preflight

- [ ] Confirm the worktree contains only intended Alpha release changes.
- [ ] Run the full local verifier:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify-alpha.ps1
```

- [ ] Confirm `scripts/verify-alpha.ps1` passed Python tests, static checks,
  JavaScript/npm, C#/NuGet, Go, Java Maven, Kotlin Maven, C, C++, CMake, CTest,
  install, and installed-consumer checks.
- [ ] Confirm `docs/burndown.md` has the latest readiness percentages and any
  blockers found during verification.
- [ ] Confirm `docs/releases/v1.0.0-alpha.1.md` lists features, install notes,
  migration notes, limitations, and beta-track gaps.

## Hosted CI

- [ ] Push the release-candidate branch or open/update the release PR.
- [ ] Observe `.github/workflows/alpha-matrix.yml` on GitHub Actions.
- [ ] Run the hosted CI check from an authenticated local environment:

```powershell
gh auth login
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-hosted-ci.ps1
```

- [ ] Record the successful hosted workflow URL in `docs/burndown.md`.

## Artifact Sanity

- [ ] Confirm Python wheel and source distribution build from the repository
  root and include `py.typed`, bundled configs, schemas, examples, and
  `liblogit-viewer`.
- [x] Confirm C and C++ install/export works through CMake and that an installed
  consumer can `find_package(libLogit CONFIG REQUIRED)`.
- [x] Confirm C++ users have both a root-header vendoring path and an installed
  `<liblogit/logit.hpp>` migration path documented.
- [ ] Confirm C#, JavaScript, Java, Kotlin, and Go package metadata matches the
  Alpha artifact names in the release notes.

## Promotion

- [ ] Change `docs/releases/v1.0.0-alpha.1.md` from draft notes to final release
  notes.
- [ ] Mark `REL-08` as `Done` in `docs/burndown.md`.
- [ ] Update the latest readiness snapshot.
- [ ] Tag the release candidate after the final verifier and hosted CI evidence
  are recorded.

## Current External Gate

As of 2026-05-24, local verification passes, but this Codex environment is not
authenticated with GitHub CLI. Hosted CI still needs to be observed after the
workflow is pushed.
