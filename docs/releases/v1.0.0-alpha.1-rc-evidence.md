## Local Verification

The repeatable local release-candidate verifier passed after the Python module
split and fixed-clock fixture contract work:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify-rc-local.ps1
```

It wraps the full Alpha matrix, package metadata sanity checks, Python artifact
inspection in a temp copy of the current tree, and installed-wheel smoke
coverage in an isolated virtual environment.

Artifact inspection and installed-wheel smoke behavior are implemented in
`scripts/verify-python-wheel-smoke.py` so the local RC verifier and hosted
Python workflow use the same proof.

The verifier now also supports a strict final-release mode:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\verify-rc-local.ps1 -RequireCleanWorktree
```

That mode should be run after committing the intended release-candidate changes
and before pushing, publishing, or tagging. It fails before artifact builds if
the worktree has local edits or untracked files.
